# 开发 Mod

准备好开始为 R.E.P.O. 制作 Mod 了吗？在设置新项目时，你通常有两个主要方向可以选择。\
最佳方案完全取决于你的经验水平以及你的 Mod 实际需要实现的功能。

### 1. Unity 工作流
- [**通过修补后的 Unity 项目**](unity.md)

如果你是 Mod 新手，或者只想在不编写任何代码的情况下创建内容，这是最佳选择。使用修补后的 Unity 项目非常适合添加新物品、模型和标准资源，特别是与 REPOLib 等工具配合使用时。

### 2. HarmonyX 工作流
- [**通过 HarmonyX 项目**](harmonyx.md)

如果你的 Mod 需要改变游戏玩法或需要自定义逻辑，你需要编写一些 C# 脚本。此工作流依赖 BepInEx 和 HarmonyX 来直接修补和修改游戏代码。

## 结合两种方法
虽然我们将这两种方法分开列出，但对于较大的 Mod，**结合使用是标准做法**。大多数高级项目使用 Unity 工作流来构建和打包自定义资源，同时使用 HarmonyX 项目来处理在游戏内真正让这些资源运作的自定义 C# 代码。
